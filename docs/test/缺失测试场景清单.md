# 缺失测试场景清单

## 背景

本清单只记录与当前 `scope.md` 一致、仍值得补齐的测试。旧语义相关场景已从清单移除，不再作为待测能力维护。

## 高优先级

### 1. 语法与范围拒绝测试

需要有明确失败断言的场景：

- `MATCH (n) RETURN n`
- `OPTIONAL MATCH`
- `UNWIND`
- `CALL`
- `SKIP`
- 变长路径
- 单跳虚拟边
- 纯虚拟点
- 虚拟到虚拟路径

### 2. 虚拟边执行顺序测试

需要补齐：

- 第一跳虚拟边按“外部 -> 提取 ID -> TuGraph”执行
- 最后一跳虚拟边按“TuGraph -> 提取 ID -> 外部”执行
- `idMapping` 在第一跳按反向关联使用
- 混合边 `|` 场景拆分后结果正确合并

### 3. WHERE 下沉测试

需要补齐：

- 纯物理条件进入 TuGraph
- 纯虚拟条件进入外部过滤条件
- 物理 + 虚拟混合条件按变量归属分别下沉
- 不再保留“物理/虚拟条件只能部分下沉”的旧限制

### 4. limit 与结果集上限

需要补齐：

- 含外部查询的默认 limit 为 `1000`
- 纯 TuGraph 计划默认 limit 为 `5000`
- 显式 `LIMIT` 覆盖默认值
- 最终结果集统一收口到 `8000`

### 5. 批量执行

需要补齐：

- 多 ID 外部依赖只发起一次批量请求
- 同源同操作同过滤条件的查询被合并
- unbatch 后结果仍按原查询归属恢复

## 中优先级

### 6. 排序与 UNION

需要补齐：

- `ORDER BY` 在支持范围内行为稳定
- `UNION` / `UNION ALL` 的去重与保留重复语义正确
- 排序与 limit 在联邦结果上的最终表现符合规范

### 7. 参数化查询

需要补齐：

- 直连 TuGraph 时参数按查询出现顺序传递
- 重复参数名按出现次数保留
- 缺失参数时抛出明确异常

## 建议落点

| 测试主题 | 建议文件 |
|----------|----------|
| 范围拒绝 | `ParserTest.java` / `GrammarQueryParserCoverageTest.java` |
| 第一跳/最后一跳执行顺序 | `RewriterTest.java` / `E2ETest.java` |
| 默认 limit / 8000 上限 | `RewriterTest.java` / `E2ETest.java` |
| 批量请求 | `ExecutorTest.java` / `E2ETest.java` |

## 维护说明

- 任何新增测试都必须以 `docs/rules/scope.md` 为准
- 不要再补 `OPTIONAL MATCH`、`UNWIND`、`CALL`、`SKIP`、变长路径等旧口径正向用例
