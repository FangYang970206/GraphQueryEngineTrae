# 07 收紧只读约束与非法语句报错

## 目标

把 SDK 的只读边界改成强约束，而不是“可解析但跳过执行”。

## 影响文件

- `graph-query-engine/src/main/java/com/fangyang/federatedquery/parser/CypherASTVisitor.java`
- `graph-query-engine/src/main/java/com/fangyang/federatedquery/rewriter/QueryRewriter.java`
- `graph-query-engine/src/main/java/com/fangyang/federatedquery/sdk/GraphQuerySDK.java`
- 相关异常类与错误码映射

## 修改要求

- 对 `CREATE`、`MERGE`、`DELETE`、`DETACH DELETE`、`SET`、`REMOVE`、存储过程统一直接报错。
- 删除“解析成功但 rewriter 跳过”的兼容逻辑。
- 清理或重写所有依赖旧行为的错误消息和测试断言。

## 验收标准

- 任何写语句或存储过程都不能被当作合法查询继续流转。
- 执行计划中不再出现由非法语句导致的空计划或静默跳过行为。
