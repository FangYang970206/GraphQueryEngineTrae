# 04 修正文档测试清单与设计说明

## 目标

修正测试规划和设计思路文档中的旧语义，保证测试目标和架构思路都以 `scope.md` 为准。

## 影响文件

- `docs/test/缺失测试场景清单.md`
- `docs/idea/图联邦查询引擎思路.md`
- `docs/idea/Cypher语法范围与约束清单.md`

## 修改要求

- 删除或改写把 `SKIP` 视为可测试能力的条目。
- 删除或改写“多数据源下 `ORDER BY/LIMIT/SKIP` 应抛异常”的口径，使其与 `scope.md` 一致。
- 删除或改写“虚拟节点 `WHERE` 不下推到 TuGraph”“`ORDER BY/LIMIT` 全局内存处理”等旧设计描述。
- 将旧版范围清单文档标记为废弃、跳转到 `docs/rules/scope.md`，或直接改为同一内容。

## 验收标准

- 测试规划文档与架构思路文档不再传播旧的执行语义。
- 仓库内不存在与 `docs/rules/scope.md` 并列且内容冲突的“范围清单”文档。
