# 12 回收测试用例与覆盖口径

## 目标

把所有固化旧语义的单测、集成测试和 E2E 测试回收为 `scope.md` 口径，避免代码修正后被旧测试阻塞。

## 影响文件

- `graph-query-engine/src/test/java/com/fangyang/federatedquery/parser/ParserTest.java`
- `graph-query-engine/src/test/java/com/fangyang/federatedquery/parser/GrammarQueryParserCoverageTest.java`
- `graph-query-engine/src/test/java/com/fangyang/federatedquery/rewriter/RewriterTest.java`
- `graph-query-engine/src/test/java/com/fangyang/federatedquery/sdk/GraphQuerySDKParameterizedQueryTest.java`
- `graph-query-engine/src/test/java/com/fangyang/federatedquery/e2e/E2ETest.java`
- `graph-query-engine/src/test/java/com/fangyang/federatedquery/e2e/LcypherFullCoverageE2ETest.java`
- `graph-query-engine/src/test/java/com/fangyang/federatedquery/e2e/MissingCoverageE2ETest.java`
- 其他断言 `SKIP`、`UNWIND`、无 label 起点、`CREATE` 通过 SDK 的测试

## 修改要求

- 删除或改写所有把 `SKIP`、`UNWIND`、`OPTIONAL MATCH`、写语句、存储过程视为成功能力的测试。
- 将 `MATCH (n)` 之类无 label 起点测试改为失败断言。
- 为单跳虚拟边、纯虚拟点、虚拟到虚拟、变长路径、跨源聚合等约束补失败测试。
- 为 `ORDER BY`、`LIMIT`、批量查询、结果集上限补充与 `scope.md` 一致的正反向测试。

## 验收标准

- 测试集不再把旧语义当成正确行为。
- `scope.md` 中的关键边界都有对应测试覆盖。
