# 01 统一核心规范文档

## 目标

以 `docs/rules/scope.md` 为唯一准入规范，回收仓库中与其冲突的“硬规则”描述，避免后续开发同时参考多套语义。

## 影响文件

- `AGENTS.md`
- `CLAUDE.md`

## 修改要求

- 将语法支持范围改为与 `scope.md` 完全一致。
- 删除或改写对 `SKIP` 的支持描述。
- 删除或改写“虚拟节点条件不下推到 TuGraph”的描述，统一为 `scope.md` 的下沉口径。
- 删除或改写“`ORDER BY`/`LIMIT` 在聚合后内存处理”的描述，统一为 `scope.md` 的下沉口径。
- 将只读范围收敛到 `scope.md` 指定集合，明确禁止存储过程。
- 如保留“硬规则”章节，必须逐条与 `scope.md` 对齐，不能再出现额外能力。

## 验收标准

- `AGENTS.md` 与 `CLAUDE.md` 中不再出现与 `scope.md` 冲突的能力说明。
- 团队在仓库根目录只会看到一套一致的语义口径。
