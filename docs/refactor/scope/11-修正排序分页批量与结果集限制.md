# 11 修正排序分页批量与结果集限制

## 目标

收敛排序、分页、批量请求和结果集上限，实现 `scope.md` 的执行约束。

## 影响文件

- `graph-query-engine/src/main/java/com/fangyang/federatedquery/rewriter/QueryRewriter.java`
- `graph-query-engine/src/main/java/com/fangyang/federatedquery/aggregator/GlobalSorter.java`
- `graph-query-engine/src/main/java/com/fangyang/federatedquery/sdk/GraphQuerySDK.java`
- `graph-query-engine/src/main/java/com/fangyang/federatedquery/executor/BatchingStrategy.java`
- `graph-query-engine/src/main/java/com/fangyang/federatedquery/plan/GlobalContext.java`

## 修改要求

- 去掉 `SKIP` 支持链路。
- 将 `ORDER BY`、`LIMIT` 的规划和执行位置调整为 `scope.md` 规定的方式。
- 为不同数据源实现默认 limit：`druid/alarm=1000`，`tugraph=5000`。
- 增加最终总结果不超过 `8000` 的统一收口逻辑。
- 将外部批量请求行为调整为符合 `scope.md` 的要求，不能继续保留与文案相反的“按数量分支”和切块语义，除非同步修订规范。

## 验收标准

- `SKIP` 查询直接报错。
- 默认 limit 和总上限能够在执行链中被观察到。
- 批量请求行为与 `scope.md` 描述一致。
