# 结果集异常语义重构 - 规格约束

## 1. 执行语义约束

### 1.1 全成功或全失败

**约束编号**: EXEC-001  
**描述**: 实际执行失败必须导致整次查询失败，不允许返回部分错误结果  
**级别**: 强制  
**验证方式**: 代码审查 + 单元测试

**详细说明**:

- 物理查询、外部查询、批量查询、Union 子查询中的任一真实失败，整次查询必须失败。
- 不允许把失败继续封装为 `QueryResult.error()` 或 warning 返回给 SDK。
- “真实失败”不包括业务上的空结果。

### 1.2 依赖查询无输入

**约束编号**: EXEC-002  
**描述**: 依赖型外部查询在无上游输入 ID 时，按空结果处理  
**级别**: 强制  
**验证方式**: 代码审查 + 集成测试

**详细说明**:

- 上游物理查询为空时，依赖查询可以不执行。
- `ExternalQuery.isReadyToExecute()` 为 false 且原因是没有输入 ID 时，不应抛异常。
- 此场景属于业务空结果，不属于执行错误。

### 1.3 最终失败传播

**约束编号**: EXEC-003  
**描述**: 并发查询失败后，整次查询必须最终失败并向上游传播  
**级别**: 强制  
**验证方式**: 代码审查

**详细说明**:

- 允许并行任务继续完成收尾。
- 不允许在主流程中吞掉失败并继续返回成功结果。
- 文档中的“终止”指最终整体失败，不要求强制取消所有并行任务。

## 2. 异常约束

### 2.1 异常类型

**约束编号**: EXC-001  
**描述**: 对外暴露的失败必须是 `GraphQueryException` 或其子类  
**级别**: 强制  
**验证方式**: 编译时检查 + 代码审查

**详细说明**:

```java
// 正确
throw new GraphQueryException(ErrorCode.DATASOURCE_QUERY_ERROR, "message", cause);

// 错误
throw new RuntimeException("message");
throw new IllegalStateException("message");
```

### 2.2 异步包装解包

**约束编号**: EXC-002  
**描述**: SDK 不得将 `CompletionException` 或 `ExecutionException` 直接暴露给调用方  
**级别**: 强制  
**验证方式**: 代码审查 + 测试

**详细说明**:

- SDK 在 `join()` 边界必须解包异步包装异常。
- 若 cause 已是 `GraphQueryException`，直接透传。
- 若 cause 不是 `GraphQueryException`，转换为 `GraphQueryException(1003)`。

### 2.3 错误码使用

**约束编号**: EXC-003  
**描述**: 异常必须使用正确的错误码  
**级别**: 强制  
**验证方式**: 代码审查

**错误码映射表**:

| 场景 | 错误码 | HTTP |
|------|--------|------|
| 查询解析失败 | `1001 QUERY_PARSE_ERROR` | 400 |
| 查询重写失败 | `1002 QUERY_REWRITE_ERROR` | 400 |
| 虚拟边约束违反 | `2001 VIRTUAL_EDGE_CONSTRAINT_VIOLATION` | 400 |
| 物理查询执行失败/超时 | `3002 DATASOURCE_QUERY_ERROR` | 500 |
| 外部数据源查询失败/超时 | `3003 EXTERNAL_DATASOURCE_ERROR` | 500 |
| 数据源适配器不存在/连接失败 | `3001 DATASOURCE_CONNECTION_ERROR` | 500 |
| 批量查询执行失败/超时 | `5002 BATCH_EXECUTION_ERROR` | 500 |
| Union 子查询失败 | `5003 UNION_EXECUTION_ERROR` | 500 |
| 通用执行装配错误 | `1003 QUERY_EXECUTION_ERROR` | 500 |

### 2.4 异常信息

**约束编号**: EXC-004  
**描述**: 异常信息必须包含足够的上下文  
**级别**: 强制  
**验证方式**: 代码审查

**必须包含的信息**:

- 查询 ID
- 数据源名称（如果适用）
- 失败原因
- 原始异常信息或 cause

## 3. 日志约束

### 3.1 错误日志记录

**约束编号**: LOG-001  
**描述**: 异常抛出前必须在执行层记录错误日志  
**级别**: 强制  
**验证方式**: 代码审查 + 日志检查

### 3.2 日志内容

**约束编号**: LOG-002  
**描述**: 错误日志必须包含完整上下文  
**级别**: 强制  
**验证方式**: 代码审查

**必须包含**:

- 查询类型（Physical / External / Batch / Union）
- 查询 ID
- 数据源名称
- 错误详情

### 3.3 单点记录

**约束编号**: LOG-003  
**描述**: 同一失败只在执行层记录一次错误日志  
**级别**: 强制  
**验证方式**: 代码审查

**详细说明**:

- SDK 不再对同一失败重复 `log.error(...)`。
- 若 SDK 需要附加上下文，使用调试日志或调用方处理。

## 4. CompletableFuture 约束

### 4.1 结果模型禁止回退

**约束编号**: CF-001  
**描述**: 不得用 `exceptionally()` 把失败重新降级为普通结果对象  
**级别**: 强制  
**验证方式**: 代码审查

**禁止示例**:

```java
return future.exceptionally(e -> QueryResult.error(e.getMessage()));
```

### 4.2 超时处理

**约束编号**: CF-002  
**描述**: 超时必须转换为业务异常，不得回退为错误结果对象  
**级别**: 强制  
**验证方式**: 代码审查

### 4.3 Union 失败传播

**约束编号**: CF-003  
**描述**: Union 子查询失败不得被吞掉  
**级别**: 强制  
**验证方式**: 代码审查 + 测试

**详细说明**:

- 禁止 `try/catch` 后仅记录日志继续聚合子结果。
- 任一子查询失败，必须使 Union 整体失败。

## 5. 代码结构约束

### 5.1 方法职责

**约束编号**: STRUCT-001  
**描述**: 执行方法只负责执行、转换异常和返回成功结果  
**级别**: 强制  
**验证方式**: 代码审查

**详细说明**:

- `executePhysical()`：执行物理查询，失败抛异常。
- `executeExternal()`：执行外部查询，失败抛异常。
- `executeBatch()`：执行批量查询，失败抛异常。
- `executeUnion()`：聚合 Union 结果，失败抛异常。
- `execute()`：协调执行，不再扫描结果对象中的失败状态。

### 5.2 失败语义位置

**约束编号**: STRUCT-002  
**描述**: 失败必须在发生点转为异常，而不是等 SDK 集中扫描  
**级别**: 强制  
**验证方式**: 代码审查

## 6. 结果对象约束

### 6.1 ExecutionResult

**约束编号**: RESULT-001  
**描述**: `ExecutionResult` 不再使用 `success` 表达状态  
**级别**: 强制  
**验证方式**: 编译时检查 + 代码审查

### 6.2 QueryResult

**约束编号**: RESULT-002  
**描述**: `QueryResult` (位于 model 包) 不再承载失败和 warning 语义  
**级别**: 强制  
**验证方式**: 代码审查

**详细说明**:

- `QueryResult` 是 model 目录下的核心领域模型，用于承载查询成功后的数据。
- 不再依赖 `success`、`error`、`warnings` 作为执行失败模型。
- `warning` 语义彻底移除。
- `5001 PARTIAL_RESULT_ERROR` 不再使用。

## 7. 测试约束

### 7.1 异常测试

**约束编号**: TEST-001  
**描述**: 所有失败场景必须有异常测试  
**级别**: 强制  
**验证方式**: 测试覆盖率检查

**必须测试的场景**:

- 物理查询失败
- 外部查询失败
- 批量查询失败
- 数据源适配器不存在
- 查询超时
- Union 子查询失败
- SDK 解包异步异常

### 7.2 空结果测试

**约束编号**: TEST-002  
**描述**: 依赖查询无输入场景必须验证为空结果而非异常  
**级别**: 强制  
**验证方式**: 集成测试

### 7.3 错误码验证

**约束编号**: TEST-003  
**描述**: 异常测试必须验证错误码，必要时验证 HTTP 映射  
**级别**: 强制  
**验证方式**: 代码审查

## 8. 兼容性约束

### 8.1 执行器签名

**约束编号**: COMPAT-001  
**描述**: `FederatedExecutor.execute()` 方法签名保持不变  
**级别**: 强制  
**验证方式**: 编译时检查

### 8.2 SDK 行为

**约束编号**: COMPAT-002  
**描述**: SDK 字符串接口失败时直接抛异常，不再返回错误 JSON  
**级别**: 强制  
**验证方式**: 集成测试

**详细说明**:

- `execute()`：成功返回 JSON 字符串，失败抛异常。
- `executeRaw()`：成功返回 JSON 字符串，失败抛异常。
- `executeWithProfile()`：成功返回 JSON 字符串，失败抛异常。
- `executeRecords()`：成功返回记录列表，失败抛异常。

### 8.3 兼容性声明

**约束编号**: COMPAT-003  
**描述**: 本次为破坏性变更，不要求兼容旧错误 JSON  
**级别**: 强制  
**验证方式**: 文档审查 + 测试迁移

## 9. 禁止事项

### 9.1 禁止返回错误结果

**约束编号**: FORBID-001  
**描述**: 禁止继续使用 `QueryResult.error()` 和 `QueryResult.partial()` 承载失败  
**级别**: 强制  
**验证方式**: 代码审查

### 9.2 禁止保留 warning 主路径

**约束编号**: FORBID-002  
**描述**: 禁止新增或保留 `warnings` 作为主执行链结果语义  
**级别**: 强制  
**验证方式**: 代码审查

### 9.3 禁止静默失败

**约束编号**: FORBID-003  
**描述**: 禁止捕获异常后仅记录日志而不继续抛出  
**级别**: 强制  
**验证方式**: 代码审查

### 9.4 禁止返回旧错误 JSON

**约束编号**: FORBID-004  
**描述**: 禁止继续通过 `buildErrorResponse()` 一类路径返回 `{success:false,...}`  
**级别**: 强制  
**验证方式**: 代码审查 + 集成测试
