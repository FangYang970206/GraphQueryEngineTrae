# Cypher语法范围与约束清单

## 一、支持范围

### 1.1 子句

- `MATCH`
- `RETURN`
- `WITH`
- `WHERE`
- `ORDER BY`
- `LIMIT`
- `UNION`
- `UNION ALL`
- `USING SNAPSHOT`
- `PROJECT BY`

除以上范围外，其余子句直接视为不支持。

### 1.2 模式

| 模式 | 说明 |
|------|------|
| 节点模式 | 支持 |
| 关系模式 | 支持 |
| 多关系类型 `|` | 支持 |
| 命名路径 | 支持 |
| 变长路径 | 不支持 |

### 1.3 表达式

| 类别 | 说明 |
|------|------|
| 比较运算 | 支持 |
| `AND` / `OR` / `NOT` | 支持 |
| `IS NULL` / `IS NOT NULL` | 支持 |
| `IN` / `STARTS WITH` / `ENDS WITH` / `CONTAINS` | 支持 |
| 聚合函数 | 仅 `count` / `sum` / `avg` / `min` / `max` |

## 二、核心约束

### 2.1 起点约束

- 每个 `MATCH` 模式的起点节点必须显式带 `label`
- 不支持 `MATCH (n) RETURN n` 这种全类型起扫

### 2.2 虚拟边边界

禁止：

- 单跳虚拟边
- 纯虚拟点
- 虚拟到虚拟路径
- `[物理] -> [虚拟] -> [物理]` 三明治结构

允许：

- 第一跳虚拟
- 最后一跳虚拟

### 2.3 WHERE 下沉

- `WHERE` 按变量归属统一下沉
- 物理条件进入 TuGraph
- 虚拟条件进入外部查询
- 不再保留旧的“只允许部分下沉”的限制口径

### 2.4 执行顺序

| 场景 | 顺序 |
|------|------|
| 第一跳虚拟 | 外部 -> 提取 ID -> TuGraph |
| 最后一跳虚拟 | TuGraph -> 提取 ID -> 外部 |
| 混合边 `|` | 拆分执行，最终合并 |

`idMapping` 统一定义为“物理字段 -> 外部字段”；第一跳虚拟边按反向关联使用。

### 2.5 批量与限制

- 禁止 N+1 外部调用
- 同源同操作同过滤条件的外部查询应合并
- 默认 limit：
  - 含外部查询：`1000`
  - 纯 TuGraph：`5000`
- 最终总结果上限：`8000`

### 2.6 不支持清单

- `OPTIONAL MATCH`
- `UNWIND`
- `CALL`
- `SKIP`
- 写语句
